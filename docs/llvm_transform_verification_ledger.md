# LLVM Transform Verification Ledger

This ledger tracks O2T progress across LLVM built-in transform
families. It is intentionally capability-oriented: entries summarize the current
verification surface for a transform area without committing to every individual
LLVM optimization pattern.

## Status Key

- **Tracked**: the project has markers, constraints, or fixtures for this area.
- **Generated IR**: `cv-replay` / config search can emit inputs that exercise the
  area.
- **Source Intent**: source mining can infer a before/after intent or safety
  contract.
- **Formal Proof**: inferred intent can be lowered to a checked formal model.
- **Impl-IR Audit**: compiled pass implementation IR can be mined and checked
  against the source intent.
- **Production Gate**: there is a focused or workflow-level regression gate.

| Loop canonicalization and analysis-driven transforms | Tracked + Generated IR; Formal Proof (IndVars loop→closed-form, polynomial delta ≤ deg 3) | Canonical header, induction PHI, simple trip count, invariant op, and early exit loop shapes. **Simulation-relation loop equivalence** (`validate/loop_simulation.py`, `cv-validate-loop-simulation`, `loop_simulation_fixture`): generalizes the induction proof to loops whose state has a DIFFERENT shape (reindexed / redundant / reshaped IVs), where positional equality no longer holds. Given a simulation relation `R(s,t)`, it proves the four obligations — `R(init_B, init_A)`; `R(s,t) ⇒ guard_B(s)==guard_A(t)`; `R(s,t) ∧ guard_B ⇒ R(step_B, step_A)`; `R(s,t) ∧ ¬guard_B ⇒ result_B==result_A` — so two structurally-different loops are proved equal for ALL trip counts (e.g. a 3-state loop with a redundant accumulator vs the 2-state original). Two-sided teeth: a miscompiled loop fails `step`, an insufficient `R` fails `result`, and an unsound `R` (an IV offset that diverges under bitvector overflow) is refuted — each with a witness. **`R` is inferred AUTOMATICALLY** by Houdini (`infer_relation` / `validate_simulation_auto`): start from every component equality `s[i]==t[j]` that holds at entry, drop any the step does not preserve under the current conjunction until a fixpoint — the survivors are the strongest inductive relation in that class. The candidate set is component equalities **plus strength-reduction affine atoms** (`t_j == c*s_i + d` for an induction variable `s_i` striding by `S` and an accumulator `t_j` striding by `K`, with the coefficient `c` solving `c·S == K` — exact division when `S | K`, else the modular inverse for odd `S` — and `d` fitted from the entry values; sound because the accumulator is not the guard variable, so the relation holds over all bitvectors regardless of overflow). Non-unit strides are handled (e.g. IV `+= 2`, accumulator `+= 10` ⇒ `c = 5`). It recovers the redundant-state relation `[(0,0),(1,1),(1,2)]` with no hint; proves a real `instcombine`-in-loop equivalence by inferring the identity relation (validating a real pass with zero hand-given `R`); and proves a **strength reduction** (a loop `i*3` replaced by an accumulator `j += 3`) by inferring the affine `j == 3*i`, which pure equality cannot bridge. A genuinely inequivalent loop or a wrong stride is refuted (the non-inductive atom is dropped, so `result` fails). Positional equivalence (induction) is the `R = (s==t)` special case. Dispatched by the orchestrator as `loop-simulation`. **UNBOUNDED nested-loop equivalence, compositional** (`validate/loop_nested.py`, `cv-validate-loop-nested`, `loop_nested_fixture`): a nested loop's outer step runs an entire inner loop, so it is proved COMPOSITIONALLY — (1) prove the two inner loops define the same transition (equal init/guard/step over the enclosing-loop variables, treated as free inputs), so the inner loops are the same function; (2) abstract the inner loop as one uninterpreted function `INNER` and prove the two outer loops equivalent with that shared `INNER` (a QF_UFBV query). Loop headers are found via a dominator computation (back-edge = an edge `B→H` with `H dom B`); the latch is the dominated incoming. A transition-preserving inner-body transform is accepted; an inconsistent inner change fails the inner check (`inner-step`), an outer change fails the outer check (`outer-step`) — localizing the failure. Dispatched by the orchestrator as `loop-nested`. **UNBOUNDED multi-exit loop equivalence** (`validate/loop_multiexit.py`, `cv-validate-loop-multiexit`, `loop_multiexit_fixture`): a loop with several exit edges (a header guard plus in-body `break`s) is modeled as an ORDERED list of exits `(fire(s), result(s))` plus a continue-`step(s)`, recovered by symbolically walking the loop region and accumulating the path condition to each exit. Two such loops are proved equivalent for all trip counts by induction over (init, per-exit decision, per-exit result, step), so a transform that flips an exit condition (fails `decision`), swaps an exit value (fails `result`), or changes the body (fails `step`) is refuted with a witness. Dispatched by the orchestrator as `loop-multiexit`. **UNBOUNDED loop-rotate validation** (`validate/loop_rotate.py`, `cv-validate-loop-rotate`, `loop_rotate_fixture`): loop-rotate is a guard-MOTION transform — a `while` loop becomes a do-while with a pre-guard + bottom guard (on the next state) + lcssa exit — so a plain lockstep proof cannot express its structural off-by-one. We reconstruct a canonical guard-on-current model from the rotated CFG and SELF-VERIFY it against the emitted instructions (the bottom guard must equal the canonical guard at the stepped state; the lcssa loop value must equal the result at the stepped state), then prove that model equivalent to the original loop for all trip counts via the simulation machinery with auto-inferred relation (which handles the permuted state). Two-sided teeth: a corrupted bottom guard fails the self-check (declined, the reconstruction no longer matches the IR) and a corrupted body fails the equivalence proof (refuted). Dispatched by the orchestrator as `loop-rotate-ir`. **UNBOUNDED loop equivalence by induction** (`validate/loop_induction.py`, `cv-validate-loop-induction`, `loop_induction_fixture`): models a natural loop as a transition system over its loop-carried state — `init`, `guard(s)`, `step(s)`, `result(s)` extracted by symbolically executing one iteration — and proves two structurally-matching loops equivalent for ALL trip counts (including non-termination) by induction: equal init, equal guards, equal step under the guard, equal result on exit (no unrolling). A structure-preserving body fold by real `opt -passes=instcombine` is proved to keep the loop's value for every `n` (single- and three-state loops); an altered step is refuted (failed obligation `step`) and a wrong exit value fails `result`, each with a state witness; a non-isomorphic loop (changed state vector) is declined. Dispatched by the orchestrator as `loop-induction`. **Bounded closed-loop TV for loop-CFG transforms** (`validate/loop_cfg_ir.py`, `cv-validate-loop-cfg-ir`, `loop_cfg_ir_fixture`): for a constant-trip-count loop, `loop-rotate` and `simple-loop-unswitch` are validated by fully unrolling the original and the transformed loop (`loop-simplify,<T>,loop-unroll,simplifycfg,instsimplify` on both) and proving the acyclic forms equal for all inputs via the existing scalar/mem2reg executors — so the transform is shown to preserve the computation for that trip count, with two-sided teeth (a corrupted output refuted) and an honest bound (a non-constant trip count is not unrolled → declined `unsupported`, never falsely proved). Dispatched by the orchestrator as `loop-cfg-ir`. Closed-loop translation validation proves real `indvars` loop→closed-form rewrites **formally for all n**: a canonical counted do-while with polynomial delta `Σ c_e·i^e` (degree ≤ 3 — `c`, `i·c`, `a·i+b`, the **cubic `i·i`**, and the **quartic `i·i·i`**) is discharged by per-widening modular lemmas over the **N-ary** product idiom `trunc((zext·…·zext)/u K)` (e.g. the quartic's 4-factor `/u 8` in i35) plus an abstracted identity over the Faulhaber closed form, mod 2³² (absorbing the modular-inverse constants), with two-sided teeth and a vacuous-proof guard (`validate/closed_form.py`, `closed_form_fixture`, `translation_validation_fixture`); declined shapes fall back to the semi-formal differential, naming the blocking ops. | Degree ≥ 4 deltas, `shl`-built deltas, widenings whose divisor/guard-width is insufficient, full LoopSimplify/SCEV/LICM semantic contracts, dominance and memory side effects. |
| LICM / loop-structural | Formal Proof (hoist legality: invariance + trap-safety) | Loop-invariant operation marker and generator shape. **Deep hoist contract** (`validate/loop_structural_model.py`, `cv-validate-licm`, `loop_structural_model_fixture`): hoisting a computation into the preheader is proved behavior-preserving under the two LICM legality conditions, each with two-sided teeth — INVARIANCE (a varying operand hoisted yields a stale value → refuted with witness `i != 0`) and SAFETY (a potentially-trapping op may be hoisted only if guaranteed-to-execute or speculatable; otherwise the unconditional preheader run introduces a new trap → refuted with a witness where the original does not execute). Both dispatched by the orchestrator (`licm-source`, `licm-model`). | Sink legality, aliasing/MemorySSA-dependent hoists, loop-rotate / unswitch CFG restructuring, poison (vs trap) propagation, and multi-op interplay beyond the single-hoist model. |
| Reassociate / InstSimplify / shift folds | Formal Proof (multi-width) + Closed-loop TV (real opt) | Extended-identity library (`constraints/extended_identities.json`): add/mul/and/or/xor associativity and commutativity, absorbing/idempotent folds (and-zero, or-allones, mul-zero, xor-zero, and/or-self, sub-self-zero), shift-by-zero; all proved at i8/i16/i32/i64 via `cv-prove-identities.py`. **Closed-loop translation validation of real `opt -passes=<P>`** for any value-preserving scalar pass via the generic `validate/scalar_ir.validate_transform` + `cv-validate-scalar-tv --passes <P>` (`scalar_tv_fixture`): reassociate, early-cse, gvn, and instsimplify outputs are each proved to keep every function's returned value for all inputs (QF_BV), with TV teeth (a wrong returned value is refuted with a witness) and the sound `unsupported` boundary. ## Cross-Cutting Infrastructure

- **IR generation** covers scalar arithmetic, CFG, memory, loop, fixed vector,
  and scalable vector shapes through bounded `GeneratorConfig` knobs. A
  `compose_bits` knob threads CFG, memory, loop, and vector regions into a single
  function (scalar lifted into a vector and a lane extracted back) and emits a
  composed global at module scope, so shapes co-occur (default-off,
  byte-identical legacy output when zero). Seeds and KLEE explore composition.
  Composed IR is validated against a real `opt`/`default<O2>` pipeline via
  `scripts/validate-ir.sh`.
- **Source mining** has both heuristic and Clang AST paths, with richer intent
  graphs for selected scalar, vector, SLP, and GlobalOpt cases.
- **Intent validation** lowers supported source intents to formal obligations and
  records unsupported side conditions instead of silently accepting them.
- **Pass implementation mining** compiles pass-like source to LLVM IR, extracts
  CFG/DFG/call/operand evidence, and currently audits selected InstCombine,
  GlobalOpt, and DSE evidence flows.
- **Workflow gates** exist for production source audit and strict GlobalOpt
  coverage. Broader transform families are still mostly fixture-gated.
- **Verdict cross-check** (`o2t/meta/cross_check.py`, `cv-cross-check`,
  `cross_check_fixture`) confirms each verdict independently of the solver that
  produced it. (1) **Witness re-validation** (z3 alone, gates today): a
  refutation's model is substituted back and the obligation is confirmed
  genuinely false there (pin every variable to its witness value, assert the
  positive goal → must be unsat) — an independent query that catches a bogus
  `sat`. (2) **Second-solver cross-check** (solver-agnostic, cvc5/cvc4
  auto-detected): a proof is replayed through every available SMT-LIB2 solver and
  all must agree; without a second solver this is honestly reported `skipped`.
  Obligations come from the verifiers' OWN `*_obligation` builders (SLP pack /
  reduction, GlobalOpt, DCE cleanup, LICM invariance + safety, DSE memory — covering QF_BV /
  QF_UF / QF_ABV / QF_FP), so there is no re-derivation. The harness has its own
  teeth: a stub solver that answers `sat` to a proved obligation is caught as a
  disagreement, and a non-falsifying (bogus) witness is not confirmed. Today: 9
  obligations replayed, 7 scalar witnesses re-validated (the array witness is
  soundly skipped).
- **Parametric generalization** (`o2t/meta/parametric.py`,
  `cv-prove-parametric`, `parametric_fixture`) lifts the structural deep
  contracts off their fixed i32/n=4 instance — the gap `prove/multiwidth.py`
  explicitly left open for non-scalar domains. Each width-parametric prover now
  takes a `width` (default 32, byte-identical legacy behavior): SLP pack +
  reduction, GlobalOpt dead-initializer, DCE cleanup, LICM hoist-invariance,
  and DSE memory are re-discharged at widths {8,16,32,64}, and the
  arity-parametric SLP contracts also at n {2,4,8,16}. At every grid point both directions are
  checked — the sound contract still **proves** and its single-point corruption
  still **refutes** — so the universal claim is proof-backed at every width/arity
  and the teeth bite at each. Today: 56 grid points, all proofs held, all 40
  teeth-bearing points refuted, no failures (a failing point would be a real
  width/arity-specific soundness finding).
- **Symbolic execution of the real pass C++** (`o2t/symexec/`,
  `cv-symexec-real-pass`, `symexec_real_pass_fixture`) verifies the IMPLEMENTATION,
  not a reconstructed model. A fold is written against a "symbolic LLVM" shim
  (`symbolic_llvm.h`: every `Value` is an SMT term, builder calls build output
  terms, analysis queries — `isKnownToBeAPowerOfTwo`, `isKnownNonNegative`, … — are
  choice points). O2T compiles the genuine C++, enumerates the pass's REAL
  control-flow paths (one per query-outcome assignment), and on each rewriting path
  proves `(facts the taken branches established) ⇒ out == in` for all inputs, with
  each query grounded by its semantic precondition via the shared
  `facts/value_tracking` encoder. So a correctly-guarded fold proves and an
  under-guarded one (rewrites on a path whose facts are insufficient) is refuted
  with a witness — caught by executing the actual branch, not a pattern match. This
  is the first O2T verifier tied to the pass's real branches rather than a regex/AST
  recovery. A **KLEE-driven** variant (`o2t/symexec/klee_driver.py`,
  `klee_fold.c`, `cv-klee-symexec-pass`, `klee_symexec_fixture`) replaces the
  hand-enumeration with true symbolic execution: the analysis queries AND the input
  opcode are `klee_make_symbolic`, so KLEE forks on every feasible branch (input
  shape × guard outcomes, incl. `&&` short-circuits), writes one test per path, and
  replay (libkleeRuntest) reproduces each path for the same per-path refinement
  discharge — finding the feasible paths automatically (an under-guarded path is
  discovered and refuted with no enumeration). Needs KLEE + its matching clang
  (LLVM 16); auto-detected and reported skipped if absent, with the enumeration path
  as the always-available fallback. The remaining layer is a poison/UB refinement
  value model (refinement here is value-equality).
- **Proof meta-audit** (`o2t/meta/proof_audit.py`, `cv-audit-proofs`,
  `proof_audit_fixture`) certifies what every "proved" verdict *means*, across all
  deep contracts (SLP pack/reduction, GlobalOpt, DCE cleanup, loop-structural,
  CFG if-conversion, DSE memory). For each proved contract it checks two anti-vacuity properties:
  (1) the assumption set is jointly **satisfiable** (no contradictory guard proves
  vacuously), and (2) every **single-point mutation** of the transform — swap a lane,
  drop an alias/safety guard, flip/negate a select condition, make a reduction op
  non-associative, expose an initializer — is **refuted with a witness**. A mutant
  that still proves is a *survivor* (a teeth gap); the gate fails on any survivor or
  unsatisfiable premise. The auditor has its own two-sided teeth (it flags a
  deliberately tautological obligation and a contradictory premise set). Today: 17
  proved contracts, 29 mutations, all killed, all premises satisfiable. This also
  surfaced and pinned a witness-key inconsistency (memory reports a parsed `witness`
  dict; the other verifiers report raw `model` text) — the auditor accepts both.

   marker spelling.
